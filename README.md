# Gmail 領収書自動保存システム (Gmail Receipt to Drive)

Gmail上の特定ラベルがついたメールから領収書PDFを抽出し、Googleドライブの月別フォルダへ自動保存するGoogle Apps Scriptです。

## 1. システム概要

- **抽出**: `領収書振り分け` ラベルが付与されたメールをスキャン
- **判定**: 添付PDF の有無を確認。Receipt → Invoice → 最初のPDF の優先順で取得
- **PDF化**: 添付がない場合、メールHTML本文を画像付きPDFとして出力
- **整理**: Googleドライブ内に `年_令和X年/01_領収書/月` のフォルダ構造を自動生成し、`YYYYMMDD_企業名.pdf` の形式で保存
- **管理**: 処理済みスレッドに `領収書` ラベルを付与し、`領収書振り分け` ラベルを削除

## 2. フォルダ構造

```
親フォルダ（PARENT_FOLDER_ID）
  └── 2026_令和8年
      └── 01_領収書
          ├── 01
          │   └── 20260115_Obsidian.pdf
          ├── 02
          └── 03
              ├── 20260315_Amazon_co_jp.pdf
              └── 20260320_Google_Cloud_本文.pdf
```

## 3. セットアップ

### 3.1 前準備

1. Googleドライブに親フォルダを作成し、URLからフォルダIDをメモ
2. Gmailで `領収書振り分け` ラベルを作成

### 3.2 GASプロジェクト作成

1. [Google Apps Script](https://script.google.com/) で新規プロジェクトを作成
2. 「プロジェクトの設定 → スクリプトプロパティ」に以下を登録

| プロパティ名 | 値 | 必須 |
|---|---|---|
| `PARENT_FOLDER_ID` | GoogleドライブのフォルダID | Yes |
| `TARGET_LABEL` | 対象ラベル名（デフォルト: `領収書振り分け`） | No |
| `DONE_LABEL` | 処理済みラベル名（デフォルト: `領収書`） | No |

### 3.3 開発環境（clasp）

```bash
npm install -g @google/clasp
clasp login
clasp clone <SCRIPT_ID> --rootDir .
```

> `.clasp.json` の `rootDir` が `"."` になっていることを確認してください。

### 3.4 デプロイ

```bash
clasp push
```

## 4. 自動化の設定

GASエディタの「トリガー」から以下を設定:

| 設定項目 | 値 |
|---|---|
| 実行する関数 | `autoArchiveReceipts` |
| イベントのソース | 時間主導型 |
| タイプ | 月タイマー |
| 日 | 10日 |
| 時刻 | 午前2時〜3時 |
| エラー通知 | 毎日通知を受け取る |

## 5. 技術仕様・制限事項

| 項目 | 説明 |
|---|---|
| **二重処理防止** | スレッド単位で `領収書振り分け` ラベルを剥がすため、次回の実行対象から自動的に外れる |
| **API権限** | Gmail, Drive, Document, 外部リクエスト（画像取得）のアクセス権限が必要 |
| **クォータ制限** | Googleの実行時間制限（6分/回）あり。大量のメールがある場合は分割処理を推奨 |
| **画像埋め込み** | HTML本文PDF化時、外部画像をbase64に変換して埋め込み |
